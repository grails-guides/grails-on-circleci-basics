The `config.yml` file tells Circle CI what to do. It must be located in a folder class `.circleci/` at the top level of your project.
Circle CI uses a docker image as the container and has the ability to cache dependencies.  It will delegate to a shell command
where we execute gradle testing.

.*.circleci/config.yml*
[source, yml]
----
include::{sourceDir}/.circleci/config.yml[]
----

<1> Docker image used as a container, with the browser option.
<2> Check fresh branch of code from GitHub.
<3> Give execution permission.
<4> Delegate to script file.
